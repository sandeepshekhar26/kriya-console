# Capturing marketing screenshots

Repeatable, **automated** capture of the sellable views from a clean, PII-free, **genuinely signed**
dataset — so every shot shows real green "verified" state, never a mock. One command:

```bash
npm run capture        # → docs/screenshots/{monitor,coverage,audit,approvals,policy,budgets,identity,evidence,fleet,connections,controlplane}.png
```

## How it works

1. **The dataset** is generated by [`demo/capture/gen-capture-data.mjs`](../../demo/capture/gen-capture-data.mjs)
   into `src/demo/capture/{capture-audit,capture-approvals}.jsonl`. It mints 34 real Ed25519-signed
   receipts (matching `src/lib/verify.ts` byte-for-byte) across three governed apps (`budget-app`,
   `crm-app`, `devops-app`) — 28 app actions plus 6 governed-lane `kriya.io.*` egress/ingress ledger
   rows (EG-2 shape, params byte-shaped like `kriya-verify/fixtures/runtime-egress-ledger.jsonl`) —
   plus one **deliberately tampered** row so the Audit view proves forgery is caught. **Actors are role/team handles** (`fin-ops`, `sales-ops`, `platform-eng`) — never people —
   and params are opaque ids with **no names and no emails**. It is deterministic (fixed signer keys),
   so re-running produces the identical committed dataset. Regenerate with
   `node demo/capture/gen-capture-data.mjs`.
2. **Capture mode**: [`capture-shots.mjs`](../../demo/capture/capture-shots.mjs) spawns its own
   `KRIYA_DEMO=1` dev server and loads `http://localhost:1455/?capture=1`, which seeds the clean dataset
   (each app as its own source) and unlocks the paid views. It screenshots each view at **1440×900 @2×**
   (2880×1800) in the **default light theme**, re-proving the Control Plane to the all-green verdict
   first, then tears the server down.

> **PII rule (load-bearing):** marketing shots must show a real, signed, but **clean** state — zero
> operator names or emails. The capture dataset is built to guarantee this; never point the capture at
> the demo `src/sample/*` or `?` (non-capture) seed, which contain placeholder people-names.

## The shots (what each sells)

| File | View | What sells |
|---|---|---|
| `monitor.png` | **Monitor** | The live home: signed-receipt tail, posture (receipts, **verified vs unverified**, signers, coverage), per-app activity. The 1 unverified row is the tamper being caught. |
| `coverage.png` | **Coverage Map** | The six-lane GREEN/AMBER/GREY grid — the honest "what *isn't* recorded" answer; every state change is itself a signed `coverage.snapshot` receipt. |
| `budgets.png` | **Budgets & rate** | Per-app/per-agent/per-operator usage vs caps — the runaway-agent stop (egress-parity item B15, already shipped). |
| `identity.png` | **Identity & access** | Per-operator + per-agent attribution from the signed `actor`; RBAC roles. |
| `fleet.png` | **Fleet** | Cross-app correlation on this machine: verified/failed, distinct signers, policy coverage. |
| `policy-egress.png` | **Policy → Egress destinations** (EG-3) | The egress tier ON: host-pattern rules (allow / require-approval), unlisted-destinations **deny-by-default**, round-tripping live into the YAML the runtime's `EgressPolicy` enforces. |
| `evidence-egress.png` | **Evidence → egress rows** (EG-2/EG-3) | 3.1.3 / 3.4.2 / 3.14.6-7 computed from the seeded signed `kriya.io.*` receipts (4 allow, 1 deny, 1 approve) — with the honest governed-lane scope block in frame. |
| `audit.png` | **Audit log** | The signed-receipt table with the **tamper-flagged row** (`delete_transaction` → "signature does not match"). *"Every receipt, verified locally against its embedded key."* |
| `approvals.png` | **Approvals** | The cross-app/agent queue, **high-risk first** (delete / transfer / close / send), role-handle actors, Approve/Deny. The clearest enterprise shot. |
| `policy.png` | **Policy** | The rule editor: ordered allow / require-approval / deny, coverage suggestions, lint, decision preview. |
| `evidence.png` | **Evidence** | SOC 2 / ISO 42001 / EU AI Act control mapping with **gaps shown honestly** (integrity reads "Broken" because the tampered receipt is caught — the value prop, not a bug). |
| `connections.png` | **Connections** | The governed-MCP catalog (kriya-native / proxy / desktop) + the add-connection flow. |
| `controlplane.png` | **Control plane** | The on-prem aggregator: fleet coverage + the **all-green trustless re-verification** (signature, hash-chain, tail-truncation anchor, "Re-proved" verdict). |

## Framing

Frame the shots in **CSS on the website** (a laptop/browser mockup), not baked into the PNG — keeps the
raw stills reusable for docs and the pitch deck. The 1440×900 captures are already 16:10. See the
website `.mockup--browser` / `.mockup--laptop` components.
